## OPP-001: Opportunity Title

- **Mode:** New idea / Existing project
- **Problem:** Pain point or unmet need.
- **Target user:** Primary user or stakeholder.
- **Desired outcome:** Observable user or business result.
- **Evidence/source:** User prompt, repo finding, research finding, metric, interview, or assumption.
- **Proposed solution:** Product-level concept, not implementation tasks.
- **MVP boundary:** Smallest useful version.
- **Non-goals:** What this opportunity should not include.
- **Assumptions:** Unknowns treated as true for now.
- **Risks:** Product, technical, security, privacy, operational, or adoption risks.
- **Success signals:** Measurable or observable indicators. Mark missing baselines as unknown.
- **Recommended next skill:** `grill-with-docs` / `product-architect`.
